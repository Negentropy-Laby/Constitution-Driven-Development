# Example Session: Implementing a Product Contract Test

**Context:** The product team needs to implement the API dry-run contract promised by the CSV import CDD and ADR. This example mirrors a game implementation session, but for product contract behavior.

**Starting State:**
- `design/cdd/csv-import.md` — acceptance criteria require stable JSON output
- `docs/architecture/adr-0001-import-contract.md` — Accepted
- `production/epics/csv-import/story-003-api-dry-run-contract.md` — Status: ready-for-dev
- `tests/unit/test_example.py` — generated by `/test-setup`
- `tests/integration/` — empty

**Session Duration:** ~35 minutes
**Turns:** 9
**Outcome:** API dry-run contract implemented with unit and integration tests

---

## Turn 1: User Starts the Story

**User:**
> /dev-story production/epics/csv-import/story-003-api-dry-run-contract.md

## Turn 2: Agent Reads the Contract

**Agent (lead-programmer):**
> I loaded:
> - Story: `production/epics/csv-import/story-003-api-dry-run-contract.md`
> - CDD: `design/cdd/csv-import.md`
> - ADR: `docs/architecture/adr-0001-import-contract.md`
> - Control manifest: current
>
> The contract requires:
> - `POST /imports/csv:dry-run`
> - no database writes
> - response fields: `rows_seen`, `rows_valid`, `rows_invalid`, `errors`, `would_write`
> - invalid rows return HTTP 422 with the same body shape
>
> Proposed files:
> - `src/cdd_import/api.py`
> - `src/cdd_import/contracts.py`
> - `tests/unit/test_import_contract.py`
> - `tests/integration/test_api_dry_run_contract.py`
>
> May I implement this structure?

**User:**
> Yes.

---

## Turn 3: Agent Flags One Ambiguity

**Agent:**
> Ambiguity: Should invalid CSV return `200` with `rows_invalid > 0`, or `422`?
>
> ADR says invalid rows are rejected at API boundary, so I recommend `422` while preserving the summary body.

**User:**
> Use 422.

**Agent writes:** implementation and tests.

---

## Turn 4: Test Evidence

**Agent:**
> Running:
> - `pytest tests/unit/test_import_contract.py`
> - `pytest tests/integration/test_api_dry_run_contract.py`
>
> Results: PASS.
>
> Evidence:
> - Valid file: 200, `would_write: false`
> - Invalid file: 422, same response shape, row-level errors present
> - Database spy confirms no write transaction starts during dry-run
